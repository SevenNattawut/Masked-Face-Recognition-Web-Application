Getting Started

Introduction

Welcome to the user manual for Real-Time Masked Face Recognition with Machine Learning on the Web Application. This guide is designed to help you make the most of our powerful and easy-to-use application, whether you're a first-time user or an experienced administrator. In this manual, you will find detailed instructions and helpful tips for using the application's many features and functionalities, as well as troubleshooting advice for common issues. Whether you're looking to learn more about the application's basic features or advanced admin capabilities, this user manual has everything you need to get started.

System Requirement

  • Recommended Hardwares (Client): Any computer with camera accessed.
  • Recommended Hardwares (Server): Computer with i5-8300H 2.3GHz processor, 8 GB of RAM, and GTX 1050 graphic card (Minimum spec)
  • Recommended Web Browsers: Microsoft Edge, Google Chrome, Mozilla Firefox
  • Recommended Screen Setting: For a better user experiencewith, webpage with 1920x938 display size works best for this application, especially on a page with live video. (Due to the server-side video)

Advanced Features and Functionality

This subsection focuses on engineering concepts and theories of Machine Learning, Deep Learning, Convolutional Neural Network, database, web application, and etc, which are the core knowledges behind this application.
Machine Learning
Machine learning is a technology in the data science field. The machine learning algorithms are used to make a prediction or classification based on data used to train the models. Additionally, the algorithms are able to learn and improve their accuracy over time. Traditional machine learning can be categorized into 4 types: supervised learning, unsupervised learning, semi-supervised learning, and reinforcement learning.
Deep Learning
Deep learning is a subfield of machine learning. It is a neural network with many layers of interconnected nodes which enable it to learn from a huge amount of data. The more layers it has, the more potential refined and accurate the prediction is. Moreover, deep learning could process unstructured data such as digital images, and could detect important features used to predict or classify the output without requiring a human interaction, unlike traditional machine learning.
Some examples of neural networks include: convolutional neural network (CNN); used mainly in image classification, and transfer learning; used for training a pre-trained model for a new objective
Convolutional Neural Network
Convolutional neural network (CNN) is a network architecture or an algorithm for deep learning. It could recognize objects or faces from the input images by learning the features of the image over many different layers, or hidden layers. CNN has three main layers which are; convolutional layer, pooling layer, and fully connected layer.
CNN Diagram
The convolutional layer will extract the features from the input image by using a kernel or a filter to move around the input until a feature map is completed. Next, the pooling layer will reduce the number of parameters in the input, which helps with training time and efficiency. Normally, pooling layers will be used after convolutional layers before passing the input to fully-connected layers. Finally, the fully-connected layer will handle the classification using the softmax function to determine what the output should be.
The Convolution Operation
Convolutional layers, a kernel which is an array of weights, will be applied to the input image to calculate a dot product between them. After that, the result will be passed into the output array. The process will be repeated by moving the filter across the input, until all dot products have been computed, thus completing a feature map. There are three parameters affecting the output size; a number of kernels, stride values, and a padding.
Max Pooling Operation
For pooling layers, a similar filter is applied to the input, but the difference is that the filter does not have weights. Pooling reduces the spatial size of the input, which results in a simpler computation due to fewer parameters to consider. The layers are also capable of extracting notable features. There are two types of pooling; max pooling which takes the highest value of the area covered by the filter, and average pooling which returns the average value.
Lastly, the fully-connected layers are used mostly at the end of the network. They use the features learned from previous layers to make predictions or perform classification. Softmax function is used to classify the input in the form of probabilities, which sum up to 1.
Transfer Learning
Transfer learning is a technique which is training an existing model already trained for one task to be able to use it on different similar tasks. The knowledge from the original model will be passed on to the new model, reducing the duration of the model learning process. Moreover, it could improve the model performance without requiring a lot of data, since the model has already known some patterns at the beginning.
The technique is suitable for a model which takes an image data as an input, and must determine what is shown in the input. The original model could be trained with a large amount of images and could make predictions on many classes. Therefore, using transfer learning would help with the modeling process, since the new model will have learned about different features already.
Face Recognition
Face recognition is a method used to verify a person's identity, using only their face as an input to compare with face images in the database. It transforms the input image into several numerical data, finds similarity from data stored, then returns the output after analyzing. The face recognition requires many conditions to make it accurate, however, if being worked on consistently, it would gradually improve.
This technology is used in many fields and areas, mostly for a security purpose. For example, law enforcement compares mugshots with images in the facial recognition system to confirm the arrested person's identity, or a corporation using face scan to record employee attendance for working days
Computer Vision
Computer vision is a technology in the artificial intelligence field. It focuses on enabling computers to have an ability to detect and identify objects, similar to human vision. For computers to replicate this capability, thousands of labeled visual data and algorithms for pattern detection are required. In the present, it is integrated in several applications; facial recognition and augmented reality for example. One of the algorithm that is used in this application is Haar Cascade.
Integral Image and Haar Features
Haar feature-based cascade classifiers, or Haar cascades for short, is an algorithm for object detection using line and edge features. In order to create a classifier, a huge number of positive and negative images is needed. Haar features are computed by finding the difference between sums of pixels in black and white rectangles. However, it would take a long time to compute if the image is very large, so an integral image is used to speed up the process by reducing the number of computations. After that, the features that will be used in a classifier are selected by a boosting technique called Adaboost.
Web Application
Web application is an application program that is stored on a server, and delivered by the internet. In order to access and use the web application, the user must use any web browser, such as Microsoft Edge, Google Chrome, or Safari, to access the application.
There are three main parts of the application: Web server, application server, and the database. The web servers manage the requests that come from a client site application, while the application server completes the requested task. And the database is used to store any needed information.
The main advantages of web applications are: the user does not require to download the application to the computer, which helps save up some storage space. The application can be accessed via various types of browsers and devices. And it does not limit the amount of users who use the application.
Database
Database is the organized collection of the data that are connected together and stored in the computer system. It is controlled by Database Management System or DBMS. The DBMS and the application are referred to as the database system. Nowadays, there are many types of databases depending on how they store the data, for example, relational database, distributed database, hierarchical database,network database, object-oriented database, and NoSQL database.

Admin Manual
This section focuses on the detail and how to use each features that are exclusive for admin users, for example, data adding, data editing, data deleting, timeline data accessing, additional data management, model training or updating, and committing log data accessing.

Admin Menu
Admin users access more functions in the application after logging in.

User Management

Admin users can add the new user data into the system, access and edit the selected user data, including updating new face images, and delete the selected user data from the system. Its scenario is similar when we go and make a new ID card or bank account, normal users only need to capture their face images, meanwhile, admin users input their information to the form on the webpage.

/newUser page
In /newUser webpage, or user adding page, is where admin users can input and add a new user information into the database. On the left side, there is a camera screen with a button set for capturing images. On the right column, there is an input form to input a new user data. Please take note that email must be unique from other existing users.

Capturing face images step by step

  
Opening face image modal
When choosing image category and press capture button, the application will automatically capture 100 images using face detection algorithm and crop them, then store in a JSON in base64 string format. When both masked and unmasked images are captured, The face image modal is accessable, when opening it, 200 images of both masked and unmasked faces are displayed in modal, separated by masked/unmasked button.

User data form between normal user and admin user
On the form, there is one main difference betweeb admin and non-admin information slot. Form for admin user needs password input, it shows when the admin checkbox is checked.

/userList page

Masked and Unmasked face images modal

Confirmation Modal before Deleting the Selected User Data
Admin users can access /userList page to access the list of existing users in the database. The information will be shown in table format, except password information. Admin can click on image preview button to see face images of the selected user. There are also 2 buttons on the right columns of each person: Edit, and Delete. When selecting edit button, it will send admin user to /editUser page, and deleting button will be displayed the confirming modal to make confirm data deleting.

/editUser page
When opening /editUser/<userId> page, the application queries the old information of the selected user and display in input slot. The method is similar to user adding page, except admin user only needs to input the updated data, including updating face images.

Timeline Data Displaying


/timeline page

Detailed information of known and unknown users
Every detected users from face recognition page are recorded in the system, and admin users can access this page to see the timeline data of detected users. There is a button to see the detail of each record, including the cropped image of detected user. Unknown users information are recorded with "UNKNOWN" values as there is no matched user data in the database, but the application still shows the cropped image, so admin users can determine who appeared in the camera scene.

Model Training or Updating

In admin menu, there is a button to click to train or update 3 face recornition models. When clicking, the server starts model training program in the background, so the application can use normally. The training duration is varied depend on the amount of users in the database, so we recommend you to train the model after worktime, and leave the server online all night.

Committing Log


Committing Log Modal
After clicking "User action log" button, the application queries the committing log data and shows in the modal. Every actions that admin users commit to the application and database are recorded in the database. This data is used to check for the change in the application, in case that there is something wrong in the database and models.

Additional Data Management


/option page
After clicking "Data Management" button, application will send user to the data management page where user can manage the position, department, and FAQ data.

Troubleshooting

If there is any error in the application, try refreshing the webpage. If the camera is not working, please check the camera connection and access permission, and restart the server application. If the server is closed unexpectedly, try checking the server terminal. And if the database is corrupted, try backing up the old /mysql directory, create a new /mysql directory, copy the files from /backup and paste in the new /mysql, then copy the database directory, as well as ibdata1 file, and paste in the new /mysql. And try restart the server. For a video about how to repair the database: [How to fix damaged database]
If you have any other problems, please contact us.
